Target 정의 — 가장 본질적인 결정

무엇을 "위험"으로 정의할지에 따라 모델의 정체성이 달라집니다. Withdrawn만? Fail만? 둘 다(struggling)?

Cohort 내 결과 분포 (week5 기준)
총 27,244명 중 Withdrawn 4,815 / Fail 7,044 / Pass 12,361 / Distinction 3,024
Cutoff 시점 행동 비교 — 결과별 평균
week5 기준. 중요한 관찰: cutoff 시점 활동량 기준으로 Fail이 Withdrawn보다 더 낮음.
해석. cutoff 시점 평균 클릭 수: Distinction 633 → Pass 459 → Withdrawn 320 → Fail 248. Fail 학생도 Withdrawn 학생만큼, 또는 그보다 더 낮은 활동 패턴을 보임. 교수자 개입 관점에서는 Fail과 Withdrawn을 함께 "학업 위험군"으로 묶는 Target C (Withdrawn + Fail) 도 검토할 가치가 있음.
Target 정의별 positive 비율
A: Withdrawn만 / B: Fail만 / C: Struggling (Withdrawn + Fail)
⚠ Target C 선택 시 두 가지 주의사항
프로젝트명 Target C(W+F)를 선택하면 엄밀히는 "조기 이탈 예측"이 아니라 "조기 학업 위험군 예측"이 됩니다. 제목·문서·논문 표현에서 명확히 해야 합니다.
새 컬럼 필요 현재 cohort CSV의 target_withdrawn은 Withdrawn만 positive로 정의된 컬럼. Target C 선택 시 새 컬럼을 만들어야 합니다. 
df["target_struggling"] = df["final_result"].isin(["Withdrawn", "Fail"]).astype(int)

§ 04  ·  CONTEXT

Feature 신호 강도

어떤 feature가 실제로 withdrawal과 관련 있는지 확인. 단순 임계값으로는 부족한 영역과 강한 신호 영역을 구분합니다.

클릭 수 구간별 Withdrawn 비율 (week5 cohort)
활동 0인 학생도 비탈퇴가 더 많음. 클릭 수만으로는 단순한 신호가 안 됨.
관찰. 클릭 200건 이하는 구간에 상관없이 ~20% withdrawn 비율로 평탄. 500건 초과 구간에서만 13.1%로 떨어짐. 즉 "많이 클릭하는 학생만 안전"이라는 비선형 관계. 이런 패턴은 단순 logistic보다 tree 계열 모델에 유리할 가능성.
과제 제출 건수별 Withdrawn 비율 (week10 cohort)
매우 깨끗한 단조 감소 — 과제 제출은 강한 단일 신호
관찰. 과제 0건 23.4% → 1건 15.9% → 2건 14.2% → 3건 12.4% → 4건+ 9.0%. 깨끗한 단조 관계. 과제 제출 횟수는 가장 신뢰할 수 있는 단일 feature가 될 가능성이 높음.
Module별 Withdrawn 비율 (week5 cohort)
모듈 간 차이가 매우 큼. GGG 7.7% → CCC 27.6%
관찰. Module 자체가 매우 강한 신호. CCC와 DDD가 위험 모듈, GGG/EEE/AAA가 안전 모듈. Module 변수는 categorical encoding 시 신중히 다뤄야 할 핵심 feature.

§ 07  ·  CLARIFICATION

참조 무결성 — 해석 정정

data_integrity_report의 right_only 검사 결과를 정확히 해석하면, "로그에 있는데 메타에 없는" 문제는 없습니다.

항목 수 의미
VLE 활동 없는 base 학생 3,365 student_info에는 있으나 student_vle에 한 줄도 없음
사용 기록 없는 assessment_id 18 assessments 메타에 정의되어 있으나 student_assessment에 등장 안 함
사용 기록 없는 site_id 96 vle 메타에 정의되어 있으나 student_vle에 등장 안 함
핵심. left_only는 모든 검사에서 0. 즉 로그에 있는데 메타에 없는 ID는 없음. 메타데이터 join에서 row 유실 문제는 발생하지 않음. 3,365명은 cohort에 남아 VLE feature가 0/결측으로 처리됨 (자동 제외 아님).
